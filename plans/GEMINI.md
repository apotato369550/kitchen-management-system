# Implementation Plans Context

## Overview
This directory contains the architectural roadmap and step-by-step implementation guides for the project. These files serve as the "source of truth" for *what* is being built and *why*.

## Active Plans

*   **`01-database-schema.md`**: Defines the initial data structure (Models, Fields, Relationships).
*   **`02-django-init.md`**: Setup guide for the Django project structure.
*   **`03-authentication-system.md`**: Specifications for the Admin vs. Management role system.
*   **`04-raw-materials-production-tracker.md`**: (Pending/In-Progress) UI/UX details for the daily tracking features.
*   **`05-purchase-order-tracker.md`**: (Pending/In-Progress) UI/UX details for the order management system.
*   **`06-remove-tutorial-feature.md`**: Record of the decision to simplify the app by removing tutorial overlays.
*   **`07-export-data-feature.md`**: Specifications for the Excel/PDF export system.
*   **`08-test-data-operations-script.md`**: Documentation for the custom CLI testing tool.

## Usage
When starting a new feature or refactoring, consult the relevant plan file first. If a deviation is necessary, these files should be updated or a new plan created to document the change in direction.
